# Sprint 132 — Stream Endpoint (`/EmbyStreams/stream`)

**Version:** v3.3 | **Status:** Complete (token update pending) | **Risk:** HIGH | **Depends:** Sprint 131

---

## Overview

New `StreamEndpointService.cs` providing a Safari-compliant proxy endpoint. This is the foundation for the unified design spec: all playback flows through this single proxy. Replaces `SignedStreamService` and `StreamProxyService` from the old architecture.

**Endpoint:** `GET/HEAD /EmbyStreams/stream?token=<stream_token>&url=<encoded-debrid-url>`

**Token model (updated):**
- Stream tokens are short-lived HMAC-signed tokens (4h expiry), NOT raw PluginSecret
- Tokens are generated by the resolver (`GenerateStreamToken`) per m3u8 entry
- Tokens are also generated per HLS segment during rewriting
- Validation: `StreamUrlSigner.ValidateStreamToken(token, secret)` — checks HMAC + expiry
- PluginSecret never appears in .strm files or m3u8 manifests

**Key capabilities:**
- Token authentication via HMAC-SHA256 stream tokens (4h TTL)
- Safari `Range: bytes=0-1` probe returns `206 Partial Content` with exactly 2 bytes
- `Accept-Ranges: bytes` on every response
- Strips `Content-Encoding` from upstream (never gzip video)
- Proper `Content-Range`/`Content-Length` for partial ranges
- Header forwarding: `Content-Type`, `Last-Modified`, `ETag`, `Cache-Control`
- HLS segment rewriting: relative URLs → absolute `/EmbyStreams/stream` URLs with **fresh stream token per segment**
- Binary streaming via `IResponse.OutputStream` with 64KB buffer
- HEAD support for content-length probing
- 401 on invalid/expired token, 502 on upstream failure, 504 on upstream timeout

## Sprint 132 Notes

**Files created:** 1 (`Services/StreamEndpointService.cs`)
**Files modified:** 1 (`Services/StreamUrlSigner.cs`)

**Security model:**
- PluginSecret NEVER appears in .strm files or m3u8 manifests
- Resolve tokens (365d) embedded in .strm files — long-lived but HMAC-signed, can't be forged
- Stream tokens (4h) used in m3u8 entries — short-lived, auto-expire
- HLS segments each get a fresh stream token — limits exposure window
- Constant-time HMAC comparison prevents timing oracle attacks

**Design decisions:**
- Stream tokens don't encode the URL — URL is a separate param, token just proves legitimacy
- Resolve tokens encode quality/id/idType — prevents tampering with query params
- Old `Sign`/`Verify`/`GenerateSignedUrl`/`ValidateSignature` methods KEPT for backward compat until Sprint 137

**Replaces:** `SignedStreamService` (HMAC-signed redirect), `StreamProxyService` (proxy session tokens)
