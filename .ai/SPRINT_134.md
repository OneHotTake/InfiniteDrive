# Sprint 134 — Multi-Tier Hydration Pipeline

**Version:** v3.3 | **Status:** Plan | **Risk:** MEDIUM | **Depends:** Sprint 133

---

## Overview

Update the catalog sync and materialization pipeline to write one `.strm` + `.nfo` per enabled quality tier (not just one). Change the `.strm` URL format from the old HMAC-signed `/EmbyStreams/Stream` to the new token-authenticated `/EmbyStreams/resolve`.

**Key change:** `.strm` files now contain `/EmbyStreams/resolve?token={resolve_token}&quality={tier}&id={id}&idType={type}` where `resolve_token` is a long-lived HMAC-signed token (365d) generated by `StreamUrlSigner.GenerateResolveToken()`.

---

## Phase 134A — Update .strm URL Format

### FIX-134A-01: Update `VersionMaterializer.BuildStrmUrl()`

**File:** `Services/VersionMaterializer.cs` (modify)

**What:**
1. Change URL format from `/EmbyStreams/VersionedPlay?titleId=...&slot=...&token=...` to:
   ```
   {embyBase}/EmbyStreams/resolve?token={resolve_token}&quality={tierLabel}&id={id}&idType={type}[&season=&episode=]
   ```
2. `resolve_token` = `StreamUrlSigner.GenerateResolveToken(tierLabel, id, idType, PluginSecret)`
3. Remove fallback to `CatalogSyncTask.BuildSignedStrmUrl()` — always use resolve URL
4. Remove reference to old HMAC signing format

**Token generation:**
```csharp
var token = StreamUrlSigner.GenerateResolveToken(
    slotKey, itemId, idType, config.PluginSecret);
var url = $"{config.EmbyBaseUrl.TrimEnd('/')}/EmbyStreams/resolve" +
    $"?token={Uri.EscapeDataString(token)}" +
    $"&quality={Uri.EscapeDataString(slotKey)}" +
    $"&id={Uri.EscapeDataString(itemId)}" +
    $"&idType={Uri.EscapeDataString(idType)}";
```

**Depends on:** Sprint 133 (token methods in StreamUrlSigner)

## Sprint 134 Notes

**Files modified:** ~6 (`VersionMaterializer.cs`, `CatalogSyncTask.cs`, `VersionSlotRepository.cs`, `PluginConfiguration.cs`, `SeriesPreExpansionService.cs`, config UI files)

**Risk assessment:** MEDIUM. Changes the fundamental .strm file format. Existing libraries will need re-sync after this sprint. The 4-tier collapse removes 2 slots that may have active users.

**Design decisions:**
- Resolve tokens in .strm files (365d HMAC) — PluginSecret never in .strm, tokens can't be forged
- Default install = single tier only — no multi-tier complexity for new users
- Rehydration triggered on tier changes — existing items get new .strm files automatically
- SeriesPreExpansionService migrated here because it writes episode .strm files
