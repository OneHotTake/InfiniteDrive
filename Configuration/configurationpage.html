🔐 Playback Authentication

        

          EmbyStreams uses a unique API key to authenticate playback requests. The key is automatically
          embedded in all .strm files.
        

        

          How it works:
          
    
            
  • Your API key is generated once during initial setup.
    • 
            
  • It's stored in your plugin configuration file (plaintext, same as debrid API keys).
    • 
            
  • The key is only used for local playback — not transmitted over the internet.
    • 
            
  • Invalid keys return 401, preventing unauthorized playback.
    • 
          

        

        

        

          For detailed security information, see the Security documentation.
        

      


      

        ⚠ Config is stored in plaintext (config/plugins/EmbyStreams.xml).
        Restrict filesystem access to the Emby config directory to trusted users only.
