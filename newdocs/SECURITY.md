# Security & Playback Authentication

## Overview

InfiniteDrive uses **HMAC-SHA256 signed tokens** for playback authentication. All `.strm` URLs carry a cryptographic signature generated by `PlaybackTokenService` — no plaintext API keys in URLs.

## How Authentication Works

### The Problem
When Emby plays a `.strm` file, it makes an HTTP request to the URL inside the file. This request comes from ffmpeg (transcoding) or the client directly (direct streaming) — **outside of Emby's normal authentication context**.

### The Solution
Every `.strm` file contains a URL with a time-limited HMAC signature:

```
/InfiniteDrive/Resolve?id=tt1234567&quality=hd_broad&token=base64url_signature|timestamp
```

The `PluginSecret` (32 random bytes, base64-encoded) is the HMAC-SHA256 key. The signature is computed over `{id}|{quality}|{timestamp}|{secret}` and validated with `FixedTimeEquals` to prevent timing attacks.

### Request Flow
```
.strm file
  → /InfiniteDrive/Resolve?id=tt123&quality=hd_broad&token=base64sig|1700000000
    → PlaybackTokenService.ValidateStreamToken(token, PluginSecret)
      → HMAC-SHA256 verification (FixedTimeEquals)
        → AIOStreams stream resolution
          → CDN URL (302 redirect) or HLS proxy
```

## Security Model

### What's Secure

- **HMAC signatures:** Tokens are cryptographically signed. Tampering is detected.
- **Time-limited:** Signatures expire after `SignatureValidityDays` (default: 365 days). Tokens include timestamp.
- **Fixed-time comparison:** `PlaybackTokenService` uses `CryptographicOperations.FixedTimeEquals` to prevent timing attacks.
- **Scoped permissions:** A valid token grants stream resolution only — no admin operations.
- **Fail-closed:** If `PluginSecret` is missing/empty, `/InfiniteDrive/Resolve` and `/InfiniteDrive/Stream` return **503**, not unsigned URLs.
- **Circuit breaker:** After 5 consecutive resolution failures, the plugin diverts to the secondary AIOStreams instance.

### Limitations

- **Stored on disk:** `PluginSecret` is saved in `InfiniteDrive.xml`. Filesystem access grants access to the secret.
- **Not encrypted at rest:** The secret is plaintext in the config file (same as all other Emby plugin settings).
- **No per-user auth:** Tokens authenticate requests, not users. Any device on the LAN with a valid `.strm` file can play.
- **Single secret:** All `.strm` files share one secret. Rotating generates a new key for all files.

### Why This Is Acceptable

1. **Filesystem access = game over:** If someone can read the config file, they can read debrid credentials, Emby tokens, etc.
2. **LAN-only exposure:** The playback endpoints listen on the Emby server's LAN interface. They are not exposed to WAN by default.
3. **Scoped to playback:** Even with a valid token, an attacker can only get stream URLs for content already in your catalog.
4. **Industry standard:** Jellyfin, Plex, and other media servers use similar local-only auth patterns.

## PluginSecret

### Generation
On first plugin load, `PlaybackTokenService.GenerateSecret()` creates 32 random bytes:
```csharp
var secretBytes = new byte[32];
RandomNumberGenerator.Fill(secretBytes);
var secret = Convert.ToBase64String(secretBytes);
```

### Rotation
1. **Generate:** `SetupService.RotateApiKey` performs a two-phase rotation:
   - Phase 1: Write new secret, keep old secret valid for 24 hours
   - Phase 2: Revoke old secret after grace period
2. **Impact:** ALL existing `.strm` files become invalid after the grace period. Trigger a full catalog sync to regenerate.
3. **Indicator:** `PluginSecretRotatedAt` Unix timestamp tracks rotation time.

## API Endpoints & Auth

| Endpoint | Auth Method | Permissions |
|----------|------------|-------------|
| `GET /InfiniteDrive/Resolve` | HMAC token in `token` param | Playback only |
| `GET /InfiniteDrive/Stream` | HMAC token in `token` param | Playback + HLS proxy |
| `GET /InfiniteDrive/Status` | Emby session auth | Authenticated users |
| `POST /InfiniteDrive/Trigger` | Emby admin auth | Admin only |
| `POST /InfiniteDrive/Admin/*` | Emby admin auth | Admin only |
| `GET /InfiniteDrive/Discover/*` | Emby session auth | Authenticated users |

### Rate Limiting
`RateLimiter` enforces per-client resolve limits. Excess requests return 429 with `Retry-After`.

## Best Practices

1. **Use HTTPS for remote access** — If Emby is accessible from outside the LAN, use a reverse proxy with HTTPS.
2. **Firewall Emby** — Restrict Emby's HTTP port to the local network.
3. **Rotate after incidents** — Generate a new `PluginSecret` if you suspect compromise.
4. **Quarterly rotation** — Good security hygiene for long-term deployments.
5. **Strong Emby admin passwords** — Controls who can access the plugin configuration page.

## Threat Model

| Threat | Mitigation |
|--------|------------|
| LAN attacker reads `.strm` file | HMAC signature prevents URL tampering |
| LAN attacker replays old `.strm` URL | Timestamp in token; expires after `SignatureValidityDays` |
| Attacker reads config file | Filesystem permissions; plugin secret is one of many secrets on the system |
| Remote exposure of Emby | Firewall; reverse proxy with HTTPS; Emby authentication |
| Malicious Emby admin | Emby admin has full access regardless of plugin auth |
| AIOStreams credential theft | AIOStreams handles its own credentials; plugin only stores the manifest URL |
